Here's a sample of how you can use a buffered view to display large datasets. The buffering used here is pure client side buffering. Only a minimum set of rows are rendered and as you scroll up/down these rows will be redrawn to keep the HTML footprint to a minimum. You can also use a buffered version of the store to load more rows from your backend incrementally as you scroll down. To learn more about the buffering capabilities of Ext JS, please see http://www.sencha.com/blog/first-look-at-ext-js-4-2-grid

Note that the js for the example code is not minified so it is readable. See bufferedview.js.
